Tokenization often gets described as a preprocessing step, which makes it sound incidental. In practice it is much closer to an architectural choice. Before the model has produced a single logit, the tokenizer has already decided which chunks of text are stable units and which ones are awkward composites.

That matters because the network never reasons over raw text. It reasons over token ids. If two strings that look similar to a human end up with very different token boundaries, then the model starts from different internal representations.

## What a tokenizer actually does

At a high level, a tokenizer maps text into a sequence of discrete units from a fixed vocabulary. Those units are often subwords rather than whole words, because a practical vocabulary has to balance coverage against size.

The tradeoff is subtle:

- larger units preserve more semantic structure per token
- smaller units improve coverage and reduce out-of-vocabulary failure modes
- pathological splits force the model to rebuild obvious structure from fragments

<Callout title="Working heuristic">
  A good tokenizer does not merely compress text. It preserves the kinds of regularities the downstream model can exploit cheaply.
</Callout>

## Why this matters in practice

For LLM work, tokenization leaks into almost every practical concern:

- prompt length is measured in tokens, not characters
- costs are billed per token, not per sentence
- formatting-heavy tasks depend on whitespace and punctuation splits
- multilingual behavior depends heavily on how the vocabulary allocates capacity
- specialized jargon may be easy or painful depending on whether common fragments already exist

<Aside>
  This is also why “same architecture, different tokenizer” is not a cosmetic change. It changes the effective input language the model is optimized over.
</Aside>

## Closing thought

A tokenizer is not just the thing that runs before the model. It defines the alphabet of objects the model is allowed to think in. Once you frame it that way, tokenization stops looking like boring plumbing and starts looking like part of the model itself.
